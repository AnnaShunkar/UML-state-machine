# UML-state-machine
# Пояснення станів 
- Створено — користувач обрав фільм, сеанс і місце, система сформувала бронювання, але оплата ще не підтверджена.
- Підтверджено — оплата успішно проведена, бронювання активне, квиток готовий до використання.
- Скасовано — користувач або адміністратор відмінив бронювання (можливо як до, так і після оплати).
- Використано — квиток був пред’явлений і відсканований у кінотеатрі, бронювання завершене.

# Переходи:
- створити бронювання → створює новий об’єкт.
- підтвердити оплату → переводить у стан Підтверджено.
- скасувати → можливе як до, так і після оплати.
- сканувати квиток → переводить у фінальний стан Використано.
